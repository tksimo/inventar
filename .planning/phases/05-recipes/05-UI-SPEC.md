## Layout & Interaction Contracts

### Recipes Page Layout

Primary visual anchor: the recipe cards — a vertically scrollable list occupying the full viewport below the top bar.

- Page container: padding var(--space-md), max-width none (full width)
- Top bar: flex row, align-items center, gap var(--space-sm), margin-bottom var(--space-md)
  - Page title "Recipes": --font-size-heading, --font-weight-semibold, --color-text-primary, flex:1
  - Import button: text button "Import URL", --font-size-body, --color-accent, 44px touch height, no border — secondary action
- Recipe list: single column, gap var(--space-sm) between cards
- "+" FAB: `position: fixed; right: var(--space-lg); bottom: var(--space-lg)`, 56×56px, --color-accent background, UtensilsCrossed or Plus icon 24px, white — matches established FAB pattern
- Empty state: centered vertically in remaining page height, UtensilsCrossed icon 48px --color-text-secondary
- No search/filter in Phase 5 (list is expected to be short; deferred)

### RecipeCard

- Height: minimum 64px, `display: flex; align-items: center; padding: 0 var(--space-md)`
- Background: --color-secondary; border-radius 8px; border: 1px solid --color-border
- Left: no icon — recipe name is the primary identifier
- Content: flex:1, margin 0 var(--space-sm)
  - Recipe name: --font-size-body, --font-weight-semibold, --color-text-primary
  - Ingredient count sub-label: --font-size-label, --color-text-secondary, "N ingredients"
- Right: ChevronRight icon (lucide-react), size 16, --color-text-secondary — navigation affordance
- Tap anywhere on card: opens RecipeDetail

### RecipeForm (Drawer)

- Follows ItemDrawer layout: `position: fixed; top:0; right:0; height:100vh; width:100vw` — slides in from right
- z-index: 60 (same as ItemDrawer — only one drawer open at a time)
- Entry animation: translateX(100%) → translateX(0), 200ms ease-out
- Background: --color-secondary; border-left: 1px solid --color-border on desktop (min-width 768px, width 480px)
- Header: 48px height, flex row, "New Recipe" or "Edit Recipe" title (--font-size-heading, --font-weight-semibold), close button (X, 44×44px)
- Body: flex:1, overflow-y auto, padding var(--space-md), flex column, gap var(--space-md)
  - Name field: label "Name" (label style), text input (full-width, 40px height)
  - Instructions field: label "Instructions" (label style), textarea (min-height 88px, resize:vertical)
  - Ingredients section:
    - Section label: "Ingredients" (label style), text-transform uppercase, letter-spacing 0.05em
    - List of RecipeIngredientRow items (gap var(--space-sm))
    - "Add ingredient" button: text button, full-width, --font-size-body, --color-accent, 44px height, dashed border 1px --color-border, border-radius 6px — inviting tap target
  - URL import section (when opened via Import flow): read-only URL field showing the imported source
- Footer: 64px height, border-top 1px solid --color-border, padding var(--space-md), flex row gap var(--space-sm)
  - "Save recipe": flex:1, height 40px, --color-accent bg, --font-weight-semibold — primary action; disabled until name is non-empty
  - "Delete recipe" (edit mode only): width 80px, height 40px, --color-secondary bg, --color-destructive text, 1px solid --color-border — matches ItemDrawer delete button

### RecipeIngredientRow (in RecipeForm)

- Layout: flex row, align-items center, gap var(--space-sm), min-height 44px
- Name input: flex:1, height 40px, --font-size-body — primary field
- Quantity input: width 80px, height 40px, type="number", min=0, step=0.01 — numeric keyboard on mobile
- Unit input: width 80px, height 40px, type="text", placeholder "g / ml / pcs" — free text
- Remove button: Trash2 icon, size 16, --color-destructive, 44×44px touch area
- Below name input (when auto-suggest match found):
  - Match pill: `display: inline-flex; align-items: center; gap: var(--space-xs); padding: 2px var(--space-sm); background: --color-accent; border-radius: 12px; cursor: pointer`
  - Pill text: --font-size-label (12px), --font-weight-semibold, --color-text-primary; truncated to 20 chars with ellipsis
  - Pill shows: "Link: [item name]" — tap accepts; X button on pill removes link
  - If no match or match declined: pill not shown; ingredient saved as unlinked

### RecipeDetail Layout

- Full-page view within the main content area (not a drawer — recipe detail deserves full focus)
- Back navigation: top bar back button (ChevronLeft, 44×44px), page title = recipe name (--font-size-heading, --font-weight-semibold, truncated), edit button (Pencil icon, 44×44px, --color-text-secondary) on the right
- Instructions section (if present): collapsible or always visible — visible by default, --font-size-body, --color-text-primary, --font-line-body 1.5, margin-bottom var(--space-md)
- Ingredients list: label "Ingredients" (label style), then one row per ingredient
  - Ingredient row: flex row, align-items center, min-height 44px
    - Left: ingredient name (--font-size-body, flex:1)
    - Right: quantity + unit (--font-size-label, --color-text-secondary) if set
    - Link indicator: small dot (4px, --color-accent) to the left of name if item_id is set — shows the ingredient is linked to inventory
- Action buttons (pinned above the bottom edge or at page bottom, full-width, flex column gap var(--space-sm), padding var(--space-md)):
  - "Check ingredients" button: full-width, height 48px, background --color-secondary, border 1px solid --color-border, --font-weight-semibold, --color-text-primary — secondary style (opens RecipeCheckSheet)
  - "Cook recipe" button: full-width, height 48px, background --color-accent, --font-weight-semibold, --color-text-primary — primary action (opens CookConfirmSheet)

### RecipeCheckSheet

- Position: `position: fixed; left:0; right:0; bottom:0; z-index: 65`
- Background: --color-secondary; border-top 1px solid --color-border; border-radius 16px 16px 0 0
- Entry animation: translateY(100%) → translateY(0), 250ms ease-out (matches QuickUpdateSheet)
- Max height: 70dvh (taller than QuickUpdateSheet — recipe ingredient lists can be long)
- Layout (top to bottom):
  - Drag handle: 4×32px, background --color-border, centered, margin-top var(--space-sm)
  - Header: recipe name (--font-size-heading, --font-weight-semibold, padding var(--space-md))
  - Scrollable ingredient list (flex:1, overflow-y auto, padding 0 var(--space-md)):
    - Per ingredient row: flex row, align-items center, min-height 44px, gap var(--space-sm)
      - Status icon: 20×20px — CheckCircle2 (green #34D399) for ✅, AlertTriangle (amber #FBBF24) for ⚠️, XCircle (red #EF4444) for ❌
      - Ingredient name: --font-size-body, flex:1, --color-text-primary
      - Quantity + unit: --font-size-label, --color-text-secondary, text-align right
      - Unit-mismatch note (⚠️ rows only): --font-size-label (12px), --color-text-secondary, displayed below the ingredient name as a second line: "Units differ — check manually"
  - Footer: 64px height, border-top 1px solid --color-border, padding var(--space-md), flex row gap var(--space-sm)
    - "Add missing to list" button: flex:1, height 40px, --color-accent bg, --font-weight-semibold — adds all ❌ and ⚠️ to shopping list. Disabled if no ❌ or ⚠️ exist (all ingredients available).
    - "Close" link: width auto, height 40px, padding 0 var(--space-md), transparent bg, --color-text-secondary

### CookConfirmSheet

- Position: `position: fixed; left:0; right:0; bottom:0; z-index: 65`
- Background: --color-secondary; border-top 1px solid --color-border; border-radius 16px 16px 0 0
- Entry animation: translateY(100%) → translateY(0), 250ms ease-out
- Max height: 80dvh (must accommodate full ingredient list including skipped rows, all scrollable)
- Layout (top to bottom):
  - Drag handle: 4×32px, background --color-border, centered, margin-top var(--space-sm)
  - Header: "Cook [recipe name]?" (--font-size-heading, --font-weight-semibold, padding var(--space-md)), one line, truncated
  - Scrollable ingredient list (flex:1, overflow-y auto, padding 0 var(--space-md)):
    - Matched ingredient row (item_id set or name-matched): flex row, align-items center, min-height 52px, gap var(--space-sm)
      - Ingredient name: --font-size-body, --font-weight-semibold, flex:1
      - Deduction stepper: minus button (−, 44×44px) + value display (--font-size-display 28px, --font-weight-semibold, min-width 64px, text-align center) + plus button (+, 44×44px). Pre-filled per D-12.
      - Unit label (if applicable): --font-size-label, --color-text-secondary, below value or alongside
    - Skipped ingredient row (unlinked/unmatched, D-13): flex row, align-items center, min-height 44px, opacity 0.6
      - Ingredient name: --font-size-body, --color-text-secondary (not semibold)
      - Right: "not in inventory" label: --font-size-label, --color-text-secondary
      - No stepper — row is read-only
  - Footer: 64px height, border-top 1px solid --color-border, padding var(--space-md), flex row gap var(--space-sm)
    - "Cook & deduct" button: flex:1, height 40px, --color-accent bg, --font-weight-semibold — primary action. Disabled during save.
    - "Cancel" link: width auto, height 40px, padding 0 var(--space-md), transparent bg, --color-text-secondary

### URL Import Flow

- Entry: "Import URL" text button in page top bar (or in empty state as secondary CTA)
- Input method: inline text input that slides below the top bar OR a simple prompt dialog (not a full drawer — keep friction minimal)
  - Preferred: an inline input bar below the top bar — text input (full-width, 40px height, placeholder "Paste recipe URL…") + "Import" button (--color-accent, 40px height, width auto)
  - On small screens: the input takes full width, "Import" button stacks below or appears as a send-icon button to the right
- Loading state: input disabled, spinner indicator next to "Import" button, label "Importing…"
- Success: RecipeForm opens pre-filled with extracted data; URL import bar dismisses
- Failure (D-06): toast error "Couldn't import recipe. Enter ingredients manually." — RecipeForm opens with only recipe name pre-filled (from page title if available)
- The URL input is not a page-level route — it is an ephemeral interaction within the Recipes page

### FAB Placement on Recipes Page

- "+" FAB (add recipe): `position: fixed; right: var(--space-lg); bottom: var(--space-lg)` — 56×56px, --color-accent, UtensilsCrossed or Plus icon. Matches established FAB pattern.
- No ScanFAB on the Recipes page.
- When RecipeCheckSheet or CookConfirmSheet is open: FAB hidden (`display: none`) to avoid z-index conflict.

## Accessibility Contract

- RecipeCard: `role="button"` or `<button>` wrapping card content; `aria-label="Open [recipe name]"`
- RecipeForm drawer: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to "New Recipe" / "Edit Recipe" heading
- RecipeForm backdrop: `aria-hidden="true"`
- Focus trap: when RecipeForm is open, Tab cycles within the drawer only; on close, focus returns to triggering card or FAB
- RecipeIngredientRow remove button: `aria-label="Remove [ingredient name]"`
- "Add ingredient" button: `aria-label="Add ingredient to recipe"`
- RecipeCheckSheet: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to recipe name heading
- RecipeCheckSheet backdrop: `aria-hidden="true"`
- Per-ingredient status icon: `aria-label` on the icon element — "Have enough", "Low or insufficient", "Missing" — so screen readers announce status without emoji
- Unit-mismatch note: associated with the ingredient row via `aria-describedby`
- "Add missing to list" disabled state: `aria-disabled="true"` + `aria-label="All ingredients available — nothing to add"`
- CookConfirmSheet: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to "Cook [recipe name]?" heading
- CookConfirmSheet backdrop: `aria-hidden="true"`
- Deduction stepper minus/plus: `aria-label="Decrease/Increase quantity for [ingredient name]"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow`
- Skipped ingredient rows: `aria-hidden="true"` is NOT appropriate (user needs to know what's skipped); instead: `aria-label="[ingredient name] — not in inventory, will be skipped"`
- Delete confirm: two-step within footer — each button has explicit `aria-label`; focus moves to "Yes, delete" on confirm state activation
- Toast notifications: `role="status"`, `aria-live="polite"` (existing Toast component pattern)
- FAB: `aria-label="Add new recipe"`
- Import URL input: `aria-label="Recipe URL"`, `autocomplete="url"`, `inputmode="url"`
- EmptyState: `role="status"`, `aria-live="polite"` (existing EmptyState component)
- RecipeDetail back button: `aria-label="Back to recipes"`
- RecipeDetail edit button: `aria-label="Edit [recipe name]"`
