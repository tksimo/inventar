# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Core Inventory.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 2 delivers full item CRUD with categories, locations, quantity tracking, and HA user attribution. UI deliverables: a working Inventory page with list/grid layout, a slide-in add/edit drawer, quick quantity controls, search + filter chips, a Settings page for managing categories and locations, and subtle user attribution on every row. No barcode scanning (Phase 3). No shopping list management (Phase 4).

This spec extends Phase 1's design token set (`frontend/src/index.css`) without overriding any existing custom properties. All new tokens are additive.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | CSS Modules + design tokens | CONTEXT.md carried-forward constraint |
| Preset | none | No shadcn — CSS Modules for all component styles |
| Component library | none | CONTEXT.md carried-forward: no Tailwind, no external UI library |
| Icon library | lucide-react (existing) | Phase 1 decision — already installed |
| Font | System font stack (existing) | Phase 1 token — `--font-family` |

**No shadcn init for Phase 2.** The CONTEXT.md carried-forward constraints explicitly require CSS Modules for all component styles and design tokens from `frontend/src/index.css`. Introducing shadcn would conflict with this constraint and require a token remapping exercise that is out of scope.

## Spacing Scale

All Phase 1 tokens carry forward unchanged. Phase 2 adds no new spacing tokens. Usage assignments for new components:

| Token | Value | Phase 2 Usage |
|-------|-------|---------------|
| `--space-xs` | 4px | Icon-to-label gap in quantity controls; status badge inner padding |
| `--space-sm` | 8px | Row vertical padding; chip horizontal padding; attribution text top margin |
| `--space-md` | 16px | Drawer horizontal padding; card padding; filter area top/bottom padding; form field vertical gap |
| `--space-lg` | 24px | Category section header bottom margin; drawer section vertical gap; settings list item padding |
| `--space-xl` | 32px | Drawer max-width inner content inset; page content top padding |
| `--space-2xl` | 48px | Drawer header height; mobile sticky search header height |
| `--space-3xl` | 64px | Drawer width offset from left edge on mobile (leaves list partially visible) |

Exceptions (carry-forward from Phase 1):
- Touch targets for all interactive controls (quantity buttons, cycle toggle, chips, drawer close button): minimum **44px** height/width
- Drawer width on desktop: **480px** fixed (not on scale — accommodates form label + field layout)
- Drawer width on mobile (< 768px): **100vw** full-width (slide-in from right, full screen on small viewports)

| Role | Value | Token | Usage |

| Secondary (30%) | #2C2C2E | `--color-secondary` | Cards, drawer surface, search header background, settings list background |
| Accent (10%) | #3B82F6 | `--color-accent` | Active nav (carry-forward); primary CTA button "Add Item"; active filter chip background; form submit button |
| Destructive | #EF4444 | `--color-destructive` | Delete item button in drawer; destructive confirmation text |
| Text primary | #F2F2F7 | `--color-text-primary` | Item names, form field values, drawer headings, Settings items |
| Text secondary | #8E8E93 | `--color-text-secondary` | Attribution text, placeholder text, inactive filter chip text, empty state body, notes text, form labels |
| Border | #3A3A3C | `--color-border` | Row dividers, card borders, drawer border-left on desktop, form field borders, chip borders (inactive) |

**New semantic tokens for Phase 2** (additive — add to `frontend/src/index.css` `:root` block):
```css
--color-status-have: #34D399;   /* green — stock status "Have" indicator dot */
--color-status-low: #FBBF24;    /* amber — stock status "Low" indicator dot + quantity badge */
--color-status-out: #EF4444;    /* red — same hue as destructive; stock status "Out" indicator dot */
```

**Accent reserved for:**
1. Active nav item left border indicator (carry-forward from Phase 1)
2. Primary "Add Item" CTA button (background only)
3. Active filter chip background
4. Form save/submit button ("Save Item")
5. Focus ring color on interactive form elements (`outline-color`)

Accent is NOT applied to: icons, row text, quantity numbers, status badges, or any secondary action.

**Status color usage rules:**
- Status colors appear ONLY as small indicator dots (8px circle) on list rows/cards next to the quantity display
- Status colors are NOT used for text, backgrounds, or borders on any element other than the dot
- "Out" status dot may use `--color-destructive` directly (same value as `--color-status-out`)

**Drawer overlay backdrop:** `rgba(0, 0, 0, 0.5)` — not a token, applied inline as it is a one-time overlay value.

## Component Inventory

### InventoryPage

**Structure:** Sticky search + filter header → scrollable item list/grid body

**Layout rules:**
- Mobile (< 768px): grouped list, one column. Category name as section header row. Dense list rows.
- Desktop (≥ 768px): grouped card grid, 2–3 columns (`grid-template-columns: repeat(auto-fill, minmax(240px, 1fr))`). Category name as section header spanning full row.
- Breakpoint: 768px (CSS media query — no JS involved).
- Layout switch is purely responsive — no user toggle.

**Search + filter header (sticky, `--color-secondary` background, `--color-border` bottom border):**
- Search input: full width, `--font-size-body`, placeholder "Search items…", 40px height, `--color-border` border, `--color-secondary` background, `--color-text-primary` input text, `--color-text-secondary` placeholder
- Filter chips row (below search): horizontal scroll on mobile if chips overflow. Chips shown only when a filter is active. No chip row rendered if no active filters.

**Floating action button (FAB):**
- Position: `fixed` bottom-right, `--space-lg` inset from edges
- Size: 56px × 56px, `border-radius: 50%`
- Background: `--color-accent`
- Icon: `Plus` from lucide-react, 24px, `--color-text-primary`
- Label: none (icon-only, `aria-label="Add item"`)
- Touch target: the 56px button itself meets the 44px minimum

### ItemRow (mobile list item)

**Structure:** Single row, `--space-sm` vertical padding, `--space-md` horizontal padding, `--color-border` bottom border

**Left section:** Item name (14px, 400) + secondary row: location text + status dot (if status-mode) or quantity (if exact-mode), all 12px 400 `--color-text-secondary`

**Right section:** Quantity controls (see QuantityControls component) + attribution below item name: "Updated by [name] · [relative time]", 12px 400, `--color-text-secondary`

**Tap target:** Entire row is tappable to open edit drawer (`cursor: pointer`, no underline). Quantity control buttons intercept their own tap events independently.

### ItemCard (desktop grid card)

**Structure:** `--color-secondary` background, `--color-border` border, 8px border-radius, `--space-md` padding

**Top:** Item name (14px, 600) + category label (12px 400 `--color-text-secondary`)

**Middle:** Location (12px 400 `--color-text-secondary`) + status dot or quantity (12px 600 `--color-text-primary`)

**Bottom:** Attribution text (12px 400 `--color-text-secondary`) + quantity controls right-aligned

**Tap target:** Entire card tappable to open edit drawer. Quantity controls intercept independently.

### CategorySectionHeader

**Structure:** Full-width row, `--space-sm` vertical padding, `--space-md` horizontal padding (list) or column-spanning grid row (grid)

**Text:** Category name, 12px, 600, uppercase, letter-spacing 0.05em, `--color-text-secondary`

**No border or background** — visually separates groups by text only.

### QuantityControls

**Two variants based on `quantity_mode` field:**

**Variant A — Exact count** (`quantity_mode === "exact"`):
- Layout: `−` button · count number · `+` button, inline flex
- Minus button: 36px × 36px, `--color-secondary` background, `--color-border` border, 6px border-radius, `−` glyph (not lucide icon), 14px 600
- Count: 14px 600 `--color-text-primary`, min-width 32px, text-center
- Plus button: same as minus
- Tap target expansion: buttons are 36px rendered; parent row provides ≥ 44px tap height by padding
- At quantity 0, pressing `−` sets `quantity_mode` to STATUS and `status` to "out" (auto-flip per D-03). No confirmation. No disabled state on `−` at 0.
- Count rounds to 1 decimal place if fractional; integer quantities show no decimal.

**Variant B — Status cycle** (`quantity_mode === "status"`):
- Layout: single pill button cycling Have → Low → Out → Have
- Pill: min-width 72px, 32px height, 16px border-radius, `--space-sm` horizontal padding
- Background changes per state: Have → `--color-status-have` at 20% opacity; Low → `--color-status-low` at 20% opacity; Out → `--color-destructive` at 20% opacity
- Text: state label ("Have" / "Low" / "Out"), 12px 600, color matches status dot color at 100% opacity
- Status dot: 8px circle, inline-block, margin-right 4px, full status color

### FilterChip

**Structure:** Inline pill, `--space-xs` vertical padding, `--space-sm` horizontal padding, 999px border-radius

**States:**
- Active: `--color-accent` background, `--color-text-primary` text, no border
- Inactive: transparent background, `--color-border` border, `--color-text-secondary` text (inactive chips used in filter picker dropdown only)

**Dismiss button:** `×` glyph inline after label, 16px, `--color-text-primary`, `--space-xs` left margin, tappable with 20px × 20px hit area minimum (expanded by padding)

**Filter picker trigger:** A `⊕ Filter` button styled as an inactive chip (border, `--color-text-secondary`, `Filter` label with `SlidersHorizontal` lucide icon 14px left of text). Tap opens a dropdown/popover listing all categories and locations as inactive chips to select.

### ItemDrawer

**Structure:** Fixed panel sliding in from the right

**Desktop (≥ 768px):** Width 480px, full viewport height, `--color-secondary` background, `--color-border` left border (1px). Backdrop: `rgba(0,0,0,0.5)` over list.

**Mobile (< 768px):** Width 100vw, full viewport height, `--color-secondary` background. No backdrop (full coverage).

**Animation:** `transform: translateX(100%)` → `transform: translateX(0)`, `transition: transform 200ms ease-out`. Closing reverses: 200ms ease-in.

**Header (48px height, `--color-border` bottom border):**
- Left: Drawer title "Add Item" or "Edit Item", 20px 600 `--color-text-primary`
- Right: `X` icon button (lucide `X`, 20px), 44px × 44px tap target, `aria-label="Close drawer"`

**Body (scrollable, `--space-md` padding):**
Form fields in vertical stack, `--space-md` gap between fields:

| Field | Control | Notes |
|-------|---------|-------|
| Name | Text input (required) | 40px height, full width |
| Category | Select dropdown | Options from categories API; last option "Manage categories →" links to Settings |
| Location | Select dropdown | Options from locations API; last option "Manage locations →" links to Settings |
| Quantity mode | Segmented control (2 options: "Exact count" / "Status") | Switches form section below |
| Quantity (exact mode) | Number input + unit label | `step="0.1"`, `min="0"` |
| Status (status mode) | 3-button segmented: Have / Low / Out | Active button filled with status color at 20% opacity + status color text |
| Reorder threshold | Number input (optional) | Shown only in exact mode; hidden in status mode |
| Notes | Textarea, 3 rows | Optional |

**Footer (64px height, `--color-border` top border, `--space-md` padding):**
- **Add mode:** Single primary button "Save Item" (full width, `--color-accent` background, `--color-text-primary` text, 40px height)
- **Edit mode:** "Save Item" button (left, flex-grow 1) + "Delete" button (right, fixed width 80px, `--color-destructive` text, `--color-secondary` background, `--color-border` border)

**Attribution (edit mode only, inside drawer body below form):**
- Separator: `--color-border` 1px top border, `--space-md` top margin
- Text: "Last modified by [HA username] on [absolute date and time]", 12px 400 `--color-text-secondary`

### SettingsPage

**Structure:** Page with two sections: Categories and Locations

**Section layout:** Section heading (20px 600 `--color-text-primary`) + list of items below

**List item:** `--color-secondary` background, `--color-border` border, 8px border-radius, `--space-md` padding, flex row
- Left: item name (14px 400 `--color-text-primary`)
- Right: `Pencil` icon button (lucide, 16px, `--color-text-secondary`, 44px tap target) + `Trash2` icon button (lucide, 16px, `--color-destructive`, 44px tap target)

**Add new entry:** Text input + "Add" button inline below each list. Input: 40px height, `--color-border` border, `--color-secondary` background, flex-grow 1. Button: 80px width, `--color-accent` background, "Add" label, 14px 600.

**Rename inline:** Tapping `Pencil` replaces the name text with an inline text input pre-filled with current name. Confirm with `Enter` key or a `Check` icon button. Cancel with `Escape` key or `X` icon button.

**Default categories** (shipped pre-loaded, not editable or deletable by user):
- Food & pantry
- Fridge & freezer
- Cleaning & household
- Personal care

Default category items render without the Pencil and Trash icon buttons (locked).

### DestructiveConfirmation (delete item)

**Pattern:** Inline confirmation within the drawer footer — no modal dialog, no separate page.

**Trigger:** Tapping "Delete" in the edit drawer footer.

**State change:** The footer transforms:
- "Delete" button disappears
- Confirmation row appears: "Delete [item name]?" (14px 400 `--color-text-primary`) + "Yes, delete" button (`--color-destructive` background, `--color-text-primary` text, 80px width) + "Cancel" button (`--color-secondary` background, `--color-border` border, 80px width)
- "Save Item" button remains unchanged

**No drawer close on delete confirm — drawer closes after successful API DELETE response.**

### EmptyState

**Used in:** Inventory page (no items), category-filtered view (no items match filter), search (no results)

**Structure:** Centered vertically in content area, `--space-xl` vertical padding
- Icon: `Package` from lucide-react, 48px, `--color-text-secondary`
- Heading: context-specific (see Copywriting Contract), 20px 600 `--color-text-primary`
- Body: context-specific (see Copywriting Contract), 14px 400 `--color-text-secondary`
- CTA button (add-item empty state only): "Add your first item" — `--color-accent` background, 40px height, `--space-md` horizontal padding

### LoadingState

**Used in:** Initial inventory list load, drawer save/delete in-flight

**Pattern:** Inventory list — skeleton rows. Three skeleton rows, each matching ItemRow height, with `--color-secondary` background animated shimmer (CSS `@keyframes` pulse from 60% to 90% opacity, 1.4s infinite).

**Drawer save/delete:** "Save Item" and "Delete" / "Yes, delete" buttons show disabled state (50% opacity) and `cursor: not-allowed` while the API request is in flight. No spinner — the opacity change is sufficient feedback for sub-1-second local network requests.

### ErrorState

**Used in:** Inventory list load failure, drawer save failure

**Inventory list load failure:**
- Same structure as EmptyState but with `AlertCircle` lucide icon (48px `--color-destructive`)
- Copy: see Copywriting Contract

**Drawer save failure:**
- Inline error message appears above the drawer footer: "Could not save. Check your connection and try again.", 12px 400 `--color-destructive`
- Drawer remains open. Save button re-enables.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tap FAB "+" | Opens ItemDrawer in add mode. Drawer slides in from right (200ms ease-out). |
| Tap ItemRow / ItemCard | Opens ItemDrawer in edit mode, pre-filled with item data. |
| Tap drawer close "X" | Drawer slides out (200ms ease-in). No confirmation if form is unmodified. If form is dirty (any field changed), show browser `confirm()` "Discard changes?" before closing. |
| Tap backdrop (desktop) | Same as tap drawer close "X". |
| Drawer save (add mode) | POST to `api/items`. On success: close drawer, item appears in list. On failure: inline error, drawer stays open. |
| Drawer save (edit mode) | PATCH to `api/items/{id}`. On success: close drawer, row updates in-place. On failure: inline error. |
| Tap "Delete" in drawer | Footer transitions to inline confirmation row (no close). |
| Tap "Yes, delete" | DELETE to `api/items/{id}`. On success: drawer closes, item removed from list. On failure: inline error. |
| Tap "Cancel" in delete confirmation | Footer returns to Save + Delete state. No API call. |
| Tap "−" on exact-count item | PATCH quantity − 1. At quantity 0, converts to status mode "Out" (single PATCH with `quantity_mode: "status", status: "out"`). Optimistic UI update, rollback on API error. |
| Tap "+" on exact-count item | PATCH quantity + 1. Optimistic UI update. |
| Tap cycle toggle on status item | PATCH next status. Optimistic UI update. |
| Type in search input | Debounce 200ms. Filter item list client-side by `name.includes(searchTerm)` (case-insensitive). No API call — full list loaded on mount. |
| Tap "Filter" chip trigger | Open filter picker (inline popover below header). List all categories and locations. Tap any to apply as active filter. |
| Tap dismissible filter chip "×" | Remove that filter. List re-renders with updated filter set. |
| Keyboard: Escape in search | Clear search input. |
| Keyboard: Escape in drawer | Close drawer (same as tap "X"). |
| Keyboard: Tab in drawer form | Standard browser tab order through form fields. Drawer footer buttons last. |
| Keyboard: Enter in Settings rename | Confirm rename. |
| Keyboard: Escape in Settings rename | Cancel rename. |
| Page load | Fetch `api/items`, `api/categories`, `api/locations` in parallel. Show skeleton state during fetch. |
| Settings: add category | POST `api/categories`. On success: list updates. |
| Settings: rename category | PATCH `api/categories/{id}`. Inline confirm UI. |
| Settings: delete category (non-default) | DELETE `api/categories/{id}`. Inline confirmation: "Items in this category will become uncategorised. Delete anyway?" + "Yes, delete" / "Cancel" inline below the item. |
| Settings: delete location | DELETE `api/locations/{id}`. Same inline confirmation pattern. Consequence: "Items in this location will have no location assigned." |

**Optimistic UI rule:** Quantity control changes (±1 and status cycle) are optimistic — update local state immediately, revert on API error with a brief error indicator (status dot flashes red once, 300ms, no persistent error message).

**No real-time push in Phase 2.** USER-03 "near real time on refresh" is satisfied by a full page refresh or manual reload. No WebSocket. No polling.

## HTML / Accessibility Contract

### Landmark structure (Inventory page)
- `<main>` contains the inventory page
- `<header>` inside `<main>` wraps the sticky search + filter header
- `<section>` per category group with `aria-label="[Category name]"`
- `<ul>` + `<li>` for list rows; CSS grid applied to `<ul>` on desktop
- FAB is a `<button>` with `aria-label="Add item"` and `aria-expanded` reflecting drawer state

### Drawer accessibility
- Drawer is `role="dialog"` with `aria-modal="true"` and `aria-labelledby` pointing to the drawer title `<h2>`
- When drawer opens: `focus()` is called on the first form field (Name input)
- Focus trap within drawer while open: Tab cycles through interactive elements inside drawer only
- When drawer closes: return focus to the element that triggered it (FAB or row/card)
- `Escape` key closes the drawer (same as clicking "X")

### Form fields
- Every input has an associated `<label>` (not placeholder-only labelling)
- Required fields: `aria-required="true"` on the Name input
- Error messages: `role="alert"` on inline error text; associated with field via `aria-describedby`

### Quantity controls
- Minus button: `aria-label="Decrease quantity for [item name]"`
- Plus button: `aria-label="Increase quantity for [item name]"`
- Status cycle toggle: `aria-label="Stock status for [item name]: [current state]"`, `aria-live="polite"` on status text so screen readers announce state changes

### Filter chips
- Active filter chip: `role="button"` with `aria-label="Remove filter: [filter name]"`
- Filter picker popover: `role="listbox"` with individual `role="option"` items; `aria-expanded` on trigger button

### Icons
- All purely decorative icons: `aria-hidden="true"`
- Icons that ARE the accessible label (FAB, drawer close, quantity buttons): icon is `aria-hidden="true"`, accessible name from `aria-label` on the parent button

### Color contrast
- All text combinations meet WCAG AA (4.5:1 for normal text, 3:1 for large text):
  - `#F2F2F7` on `#1C1C1E`: ~16.5:1 (passes)
  - `#F2F2F7` on `#2C2C2E`: ~14.5:1 (passes)
  - `#8E8E93` on `#1C1C1E`: ~4.6:1 (passes AA for 14px)
  - `#3B82F6` on `#1C1C1E`: ~4.5:1 (passes AA — used only for large button backgrounds with white text on top)
  - Status color text (`#34D399`, `#FBBF24`, `#EF4444`) on status pill background (same color at 20% opacity): these are 14px+ — verify at implementation; if any fail, darken the text color by 20% (e.g. `#10B981`, `#D97706`, `#DC2626`)
