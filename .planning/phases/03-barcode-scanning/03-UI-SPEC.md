## Layout & Interaction Contracts

### CameraOverlay

- Position: `position: fixed; inset: 0; z-index: 70` (above ItemDrawer at z-index 60)
- Background: rgba(0, 0, 0, 0.85) — full screen
- Scan region: centered rectangle, width = min(80vw, 320px), height = same (square), transparent cutout via border overlay or clip technique
- Aiming corners: 4 corner L-shapes, --color-accent, no full border (focus visual)
- Status text: below scan region, --color-text-primary, --font-size-body, center aligned, e.g. "Point camera at a barcode"
- Close button: top-right, 44×44px, X icon (lucide-react `X`), --color-text-primary
- Entry animation: fade in 150ms ease-out
- Exit: immediate on barcode detected (camera dismissed before bottom sheet appears); or on X tap
- Camera must have `playsinline autoplay muted` attributes — required for iOS Safari through HA ingress HTTPS

### ScanFAB

- Position: `position: fixed; right: var(--space-lg); bottom: 88px` — stacked 16px above Plus FAB (56px FAB + 24px bottom offset + 8px gap = 88px)
- Size: 56×56px, border-radius 50% — matches existing FAB
- Background: --color-secondary (visually distinct from Plus FAB which uses --color-accent)
- Icon: lucide-react `ScanBarcode` size 24, --color-text-primary
- aria-label: "Scan barcode"
- hover: opacity 0.9
- focus-visible: 2px solid --color-accent, offset 2px

### QuickUpdateSheet

- Position: `position: fixed; left: 0; right: 0; bottom: 0; z-index: 65` (below CameraOverlay, above ItemDrawer)
- Background: --color-secondary
- Border-top: 1px solid --color-border
- Border-radius: 16px 16px 0 0
- Drag handle: 4px × 32px, background --color-border, centered, margin-top: 12px
- Entry animation: `transform: translateY(100%)` → `translateY(0)`, 250ms ease-out
- Dismiss: tap outside (backdrop click), swipe down >40% of sheet height, or "Done" button
- Backdrop: rgba(0,0,0,0.5), z-index 64
- Max height: 50dvh — never covers more than half the screen
- Layout (top to bottom):
  - Drag handle (12px top margin)
  - Item name: --font-size-heading, --font-weight-semibold, --color-text-primary, padding: var(--space-md)
  - Location text: --font-size-label, --color-text-secondary, padding: 0 var(--space-md) var(--space-sm)
  - Quantity display: --font-size-display (28px), --font-weight-semibold, --color-text-primary, centered
  - QuantityControls component (existing), centered, padding: var(--space-md)
  - Footer: 64px height, border-top 1px solid --color-border, padding: var(--space-md), flex row gap: var(--space-sm)
    - "Done" button: flex:1, height 40px, --color-accent bg, --font-weight-semibold — primary action
    - "Edit item" button: width auto, height 40px, padding 0 var(--space-md), transparent bg, --color-text-secondary, --font-size-body — secondary action

### NutritionSection (inside ItemDrawer)

- Rendered only when `calories_per_100g || protein_per_100g || carbs_per_100g || fat_per_100g` is non-null
- Separator: border-top 1px solid --color-border, margin-top var(--space-md), padding-top var(--space-md)
- Section label: "Nutrition (per 100g)", --font-size-label, --font-weight-semibold, --color-text-secondary, margin-bottom var(--space-sm)
- Product image: if image_url set — display above nutrition table, width 100%, max-height 120px, object-fit: cover, border-radius 6px
- Nutrition table: 2-column grid (label | value), gap var(--space-xs)
  - Labels: --font-size-label, --color-text-secondary (e.g. "Calories", "Protein", "Carbs", "Fat")
  - Values: --font-size-body, --color-text-primary, --font-weight-semibold (e.g. "245 kcal", "8.2 g")
  - Only render rows where the value is non-null

### Dual FAB Layout

- Plus FAB: `position: fixed; right: var(--space-lg); bottom: var(--space-lg)` (unchanged — 24px from bottom)
- ScanFAB: `position: fixed; right: var(--space-lg); bottom: 88px` (56 + 24 + 8 = 88px from bottom)
- Both FABs have identical shadow: `0 4px 12px rgba(0,0,0,0.3)`
- On screens shorter than 480px height: reduce gap to 4px (ScanFAB bottom: 84px) to avoid crowding

## Interaction States

### Camera Activation

| State | Visual |
|-------|--------|
| Opening | CameraOverlay fades in (150ms), camera stream starts immediately |
| Scanning | Status text "Point camera at a barcode", animated scan line optional (1px accent line sweeping vertically, 2s loop) |
| Detected | Status text changes to "Barcode detected…", overlay fades out immediately |
| Permission denied | Overlay remains, video hidden, error message shown with Dismiss button |
| User dismisses | X button tapped — overlay fades out, camera stream stopped |

### Bottom Sheet

| State | Visual |
|-------|--------|
| Opening | Sheet slides up from translateY(100%) over 250ms ease-out; backdrop fades in |
| Idle | Quantity value updates optimistically on +/− tap; "Done" enabled at all times |
| Saving | "Done" button shows "Saving…" text, disabled; re-enables on success |
| Save error | "Done" reverts to label, small error text above footer: "Couldn't save. Try again." (--color-destructive, --font-size-label) |
| Dismissed | Sheet slides down translateY(100%) over 200ms ease-in; backdrop fades out |

### OFF Lookup (during ItemDrawer open)

| State | Visual |
|-------|--------|
| Loading | Spinner or "Looking up product…" text below barcode field in drawer body |
| Found | Fields pre-filled silently — no success toast |
| Not found | Fields left empty except barcode — no error shown |
| Network error | "Couldn't look up product. Fill in manually." — --font-size-label, --color-text-secondary below barcode field |

## Accessibility Contract

- Camera overlay close button: aria-label="Close camera"
- Camera overlay: role="dialog", aria-modal="true", aria-label="Barcode scanner"
- QuickUpdateSheet: role="dialog", aria-modal="true", aria-labelledby pointing to item name heading
- QuickUpdateSheet backdrop: aria-hidden="true"
- Backdrop click dismisses sheet (mouse and touch)
- Focus trap: when CameraOverlay or QuickUpdateSheet is open, Tab cycles within the overlay/sheet only
- On close: focus returns to the Scan FAB
- NutritionSection product image: alt="" (decorative, actual name is already shown in drawer title)
- NutritionSection table: rendered as definition list (`<dl>`) with `<dt>` for labels and `<dd>` for values — not a `<table>` (only 4 pairs, semantic dl is cleaner)
- QuantityControls −/+ buttons in bottom sheet: existing aria-label pattern maintained
